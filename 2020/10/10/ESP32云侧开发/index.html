ESP32云侧开发

ESP32云侧开发

官方步骤

环境搭建

https://docs.espressif.com/projects/esp-idf/zh_CN/latest/esp32/get-started/index.html

先进行环境配置，直接从第五步：开始创建工程开始

打开bash终端，输入如下命令，因为环境变量配置在bash中。

bash
cd ~/esp
cp -r $IDF_PATH/examples/system/ota/simple_ota_example .

Demo参考(https://blog.csdn.net/chen244798611/article/details/98121985)

添加git管理

git init
cp esp-idf/.gitignore simple_ota_example/.gitignore

编译

idf.py编译
cd simple_ota_example
idf.py menuconfig
idf.py build
make编译 (传统 GNU Make)

https://docs.espressif.com/projects/esp-idf/zh_CN/latest/esp32/get-started-legacy/index.html

使用bash，或者配置IDF_PATH

python -m pip install --user -r $IDF_PATH/requirements.txt

编译工程

make

没有menuconfig可以运行

make menuconfig

下载

云端无法连接设备下载，需要将bin文件利用cp命令通过jupter notebook下载到本地，再使用flash_download_tool进行下载。

Platformio使用

https://docs.platformio.org/en/latest/tutorials/espressif32/espidf_debugging_unit_testing_analysis.html?utm_source=docs.espressif.com

安装Platformio

使用VS Code安装Platformio IDE插件，如果需要使用PIO Remote功能，需要本地也安装，映射本地串口使用。

打开Platformio终端

使用Platformio的终端，Ctrl + Shift + P，然后输入PlatformIO:new terminal。

cd ~/esp/simple_ota_example

创建Platformio工程

pio project init -d ~/esp/simple_ota_exampleio -b esp-wrover-kit --ide vscode

配置Platformio工程

工程简单结构

project_dir
├── include
├── src
│    ├── CMakeLists.txt
│    └── main.c
├── CMakeLists.txt
└── platformio.ini

将platformio.ini文件中追加如下内容

[platformio]
src_dir = main

[env:esp-wrover-kit]
platform = espressif32
board = esp-wrover-kit
framework = espidf

upload_port = COM4
upload_speed = 115200
upload_protocol = esptool

编译工程并下载至远程设备

需要本地VS Code安装Platformio IDE插件，并且启动服务

pio remote agent start

然后在云端运行命令

pio remote run --target upload -e esp-wrover-kit

第一次会根据platformio.ini的配置安装工具，安装目录在~/.platformio。

idf路径在~/.platformio/packages/framework-espidf中。

监控远程设备

pio remote device monitor -p COM4 -b 115200
